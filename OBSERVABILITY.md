# Observability

Flow-POS exposes a small built-in observability baseline:

- `x-request-id` is accepted from clients or generated by the API.
- Each HTTP request emits a JSON access log event named `http.request`.
- `/metrics` exposes simple Prometheus-style counters for request count and cumulative latency.
- `/health` reports process liveness.
- `/health/ready` checks database connectivity and Redis when configured.

## Production Alerts

Start with alerts for:

- API health check failure.
- Readiness check failure.
- Sustained HTTP 5xx responses.
- High request latency.
- PostgreSQL disk usage.
- Failed or stuck print jobs.
- Redis connectivity failure when `REDIS_URL` is configured.

## Error Tracking

Use Sentry, OpenTelemetry, or your hosting provider's error tracking in production. Do not log passwords, refresh tokens, access tokens, or raw request bodies.
